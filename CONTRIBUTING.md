# Contributing Guide

## Branch Rules
Do not work directly on `main`.

Create a new branch for every task using one of these formats:

- `backend/<feature-name>`
- `web/<feature-name>`
- `mobile/<feature-name>`
- `docs/<topic>`
- `fix/<issue-name>`
- `chore/<task-name>`

## Examples
- `backend/auth-api`
- `web/login-page`
- `mobile/home-screen`
- `fix/navbar-bug`

## Workflow
1. Pull latest `main`
2. Create a new branch
3. Push your branch
4. Open a Pull Request into `main`
5. Wait for review
6. Merge only after approval
